## 🎰 로또 1218회 — 자동구매 실패

### 무슨 일이 있었나?

매주 수요일 15시에 자동으로 로또를 분석하고 구매하는 크론잡이 설정되어 있습니다. 추천 번호 3장 + 자동 2장, 총 5장(5,000원)을 구매하는 방식이죠.

그런데 이번 주(4월 2일 수요일), **API 과부하로 크론이 실패**했습니다.

```
Error: All models failed (3)
- anthropic/claude-sonnet: overloaded
- openai/gpt: rate_limit  
- anthropic/claude-opus: overloaded
```

세 개 모델 모두 동시에 사용 불가 상태였습니다. 결과적으로 **1218회(4/5 토요일 추첨) 로또를 구매하지 못했습니다.**

### 왜 세 모델이 동시에?

이건 단순 운이 아닙니다. 글로벌 AI 서비스들이 동시에 부하가 걸리는 시간대가 있습니다:
- 미국 오전 업무 시작 시간 (한국 밤~새벽)
- 대규모 AI 서비스 업데이트/점검 시
- 월초/분기초 사용량 급증 시기

이번 4월 초가 딱 그 시기였습니다.

### 교훈과 대응

| 문제 | 대응 |
|------|------|
| 단일 실행 의존 | 재시도 로직 추가 검토 |
| 모든 모델 동시 실패 | 로컬 LLM 폴백 검토 |
| 알림 부재 | 실패 시 즉시 텔레그램 알림 |

로또는 어차피 완전한 랜덤이지만, **자동화 시스템의 안정성**은 투자와 직결되는 문제입니다.

### 이번 주 주요 변경사항

#### 1. 크론 시스템 최종 확정

감성분석과 매매 크론을 완전히 분리했습니다:

| 크론 | 주기 | 역할 |
|------|------|------|
| 감성분석 | 4시간 (X:05) | 뉴스 수집 → Claude 분석 → 상태 저장 |
| 매매 실행 | 4시간 (X:35) | 상태 파일 읽기 → 매매 실행 |
| TP 체커 | 30분 (:20, :50) | 익절 조건 모니터링 |

핵심 개선점은 **API 의존성 완전 제거**입니다. 감성분석이 실패해도 매매는 마지막 성공 상태로 독립 실행됩니다.

#### 2. 극단 하락 시뮬레이션

이더리움의 역대 최대 하락률(-94.3%, 2018년)을 v4 전략에 대입해 시뮬레이션했습니다.

**결과 요약:**
- DCA(물타기) 자본이 약 -24% 지점에서 고갈
- 나머지 -70% 하락은 무방비 상태
- 숏 사이드의 반복 익절이 쿠션 역할
- **원금 전액 손실은 구조적으로 불가능** (양방향 헤지 특성)
- 최악 예상: -30~50% 순손실

v4는 ±20~30% 변동에 최적화된 전략이며, -94% 같은 극단 시나리오에서도 헤지 구조 덕분에 순수 롱 전략보다 훨씬 안전합니다.

#### 3. 감성 확인 기간(Sentiment Confirmation) 설계

BEARISH → BULLISH 전환 시 **3회 연속 확인 후 진입**하는 메커니즘을 설계했습니다:

- 베어마켓 랠리(가짜 반등) 함정 방지
- 헤지 전략이므로 한쪽만 차단 불가 → **라운드 시작 게이트**로 작동
- Phase 1 전용 (Phase 2 DCA는 미적용)

**현재 미적용 상태**입니다. 시장이 안정적이고 현행 전략이 잘 작동하고 있어, 필요 시 활성화할 수 있도록 설계만 완료해두었습니다.

### ETH 시장 현황

현재 ETH는 ATH($4,955) 대비 약 -60% 수준에서 거래 중입니다:
- ETH/BTC 비율: 0.032 (5년래 최저)
- 2026 Q1 수익률: -32.8% (역대 최악 분기)
- 공포탐욕지수: 극단적 공포 구간

기술적으로 교착 상태에 있으며, 큰 방향성 움직임이 임박했다는 분석이 많습니다.
