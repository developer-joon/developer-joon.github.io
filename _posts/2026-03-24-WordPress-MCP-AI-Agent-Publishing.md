2026년 3월 20일, WordPress.com이 발표한 MCP 통합은 웹 퍼블리싱 역사의 분기점이 될지도 모릅니다. AI 에이전트가 단순히 글을 돕는 것이 아니라, **직접 블로그 포스트를 작성하고 발행하는** 시대가 도래한 것입니다.

## MCP가 바꾸는 WordPress의 미래

### MCP(Model Context Protocol)란?

MCP는 Anthropic이 주도하여 개발한 **오픈 프로토콜**로, AI 에이전트가 다양한 애플리케이션과 상호작용할 수 있도록 표준화된 인터페이스를 제공합니다. USB가 하드웨어 연결을 표준화했듯, MCP는 AI와 소프트웨어 간 연결을 표준화합니다.

2025년 10월, WordPress.com은 MCP 읽기 기능을 먼저 도입했습니다. AI 에이전트가 블로그 콘텐츠, 설정, 분석 데이터를 조회할 수 있었지만, **보기만** 할 수 있었죠. 하지만 사용자들은 더 많은 것을 원했습니다. "읽는 것도 좋지만, 실제로 **뭔가 해줬으면** 좋겠어요!"

![MCP Protocol](/images/2026-03-24-WordPress-MCP-AI-Agent-Publishing/mcp-protocol.jpg)

### 19개의 새로운 능력

2026년 3월 20일 업데이트로 WordPress.com은 **쓰기 기능(Write Capabilities)**을 추가했습니다. 이제 AI 에이전트는 6개 콘텐츠 유형에 걸쳐 19개의 작업을 수행할 수 있습니다:

**📝 포스트 & 페이지**
- 블로그 포스트 작성 및 발행
- 랜딩 페이지, About 페이지 생성
- 기존 콘텐츠 수정 및 업데이트

**💬 댓글 관리**
- 댓글 승인 및 답글 작성
- 스팸 댓글 정리

**🏷️ 카테고리 & 태그**
- 카테고리 생성, 이름 변경, 계층 구조 조정
- 태그 자동 할당 및 정리

**🖼️ 미디어 메타데이터**
- 이미지 alt 텍스트, 캡션, 제목 수정
- SEO 및 접근성 개선

모든 작업은 **자연어 명령**만으로 가능합니다:

> "방금 작성한 포스트를 드래프트로 발행하고, '여행' 카테고리에 분류하고, 관련 태그 붙이고, 160자 이하 메타 설명 작성해줘."

> "미디어 라이브러리에서 alt 텍스트 빠진 이미지 찾아서 파일명 기반으로 제안해줘."

## AI가 블로그를 쓸 때 — 실전 시나리오

### 1. 레시피 블로그 자동화

```
"내 블로그에 레시피 섹션을 시작하고 싶어. 
'레시피' 카테고리를 만들고, 하위 카테고리로 
아침, 점심, 저녁, 디저트를 추가해줘."
```

AI 에이전트는 카테고리 구조를 설정하고, 첫 레시피 포스트를 드래프트로 작성합니다. 테마의 색상, 폰트, 블록 패턴을 자동으로 분석해서 **사이트 디자인과 일관성 있는** 콘텐츠를 생성합니다.

![AI Writing](/images/2026-03-24-WordPress-MCP-AI-Agent-Publishing/ai-writing.jpg)

### 2. 접근성 감사 및 자동 수정

```
"내 웹사이트 접근성 감사해서 보고서 만들어줘."
```

AI 에이전트는 사이트를 스캔하고, 누락된 alt 텍스트를 찾아 자동으로 생성 제안합니다. 사용자 승인 한 번으로 수백 개의 이미지 메타데이터가 업데이트됩니다.

### 3. 디자인 인식 콘텐츠 생성

AI 에이전트는 콘텐츠를 만들기 전에 **테마 디자인을 먼저 조사**합니다. 색상, 폰트, 간격, 블록 패턴을 파악하고, 생성할 콘텐츠가 사이트의 디자인 시스템을 자동으로 상속하도록 합니다.

테마를 변경해도 문제없습니다. AI 에이전트는 새 테마에 맞춰 자동으로 적응합니다.

## 안전장치 — 통제권은 사용자에게

WordPress.com은 AI 에이전트에게 강력한 권한을 주면서도, **다층적 보안 장치**를 마련했습니다:

### ✅ 모든 변경은 승인 필요

AI 에이전트는 어떤 작업을 수행하기 전에 **정확히 무엇을 할 것인지** 설명하고 사용자의 명시적 승인을 요청합니다. 클릭 한 번 없이는 아무것도 변경되지 않습니다.

### 📄 신규 포스트는 기본 드래프트

AI 에이전트가 포스트나 페이지를 만들면 **드래프트 상태**로 저장됩니다. 사용자가 검토하고 만족한 뒤에야 발행할 수 있습니다.

기존 발행된 포스트를 수정할 때는 "이 변경은 즉시 공개됩니다"라는 경고를 표시합니다.

### 🗑️ 삭제는 복구 가능

포스트, 페이지, 댓글, 미디어를 삭제하면 **휴지통으로 이동**하며 30일간 복구 가능합니다.

단, WordPress가 휴지통을 지원하지 않는 카테고리와 태그 삭제는 **영구 삭제**임을 명확히 경고하고 추가 확인을 요구합니다.

### 📊 모든 활동은 Activity Log에 기록

AI 에이전트가 수행한 모든 작업은 사이트의 **Activity Log**에 자동 기록됩니다. 언제든지 "어떤 변경 했어?"라고 물으면 목록을 보여줍니다.

### 🔐 WordPress 권한 시스템 준수

AI 에이전트는 WordPress의 기존 **사용자 역할 권한**을 그대로 따릅니다:
- **Editor**: 포스트 생성/수정 가능, 사이트 설정 변경 불가
- **Contributor**: 드래프트 작성 가능, 발행 불가
- **Author**: 자기가 쓴 글만 수정/삭제 가능

사용자가 할 수 없는 일은 AI 에이전트도 할 수 없습니다.

### ⚙️ 세밀한 토글 제어

MCP 설정(`wordpress.com/me/mcp`)에서 19개 작업 각각을 개별적으로 활성화/비활성화할 수 있습니다. "포스트 작성만 허용하고 삭제는 막기" 같은 세밀한 제어가 가능합니다.

![WordPress Automation](/images/2026-03-24-WordPress-MCP-AI-Agent-Publishing/wordpress-automation.jpg)

## 기존 REST API와의 차이

WordPress REST API는 이미 프로그래밍 방식으로 콘텐츠를 관리할 수 있었습니다. 그렇다면 MCP는 무엇이 다를까요?

### 자연어 인터페이스

REST API는 정확한 엔드포인트, 파라미터, 인증 토큰이 필요합니다. 개발자용입니다.

MCP는 **"레시피 카테고리 만들어줘"** 같은 자연어 명령으로 작동합니다. 비개발자도 쉽게 사용할 수 있습니다.

### 컨텍스트 인식

REST API는 요청한 것만 정확히 수행합니다.

MCP 기반 AI 에이전트는 **사이트의 디자인, 기존 카테고리 구조, SEO 규칙, 콘텐츠 스타일**을 이해하고 맥락에 맞는 작업을 수행합니다.

### 승인 기반 워크플로우

REST API는 스크립트가 실행되면 멈추지 않습니다.

MCP는 각 단계에서 **사용자 승인을 요청**하며, 사용자가 워크플로우를 제어합니다.

## 개발자/블로거에게 미치는 영향

### 개발자 관점

**WordPress 플러그인/테마 개발이 바뀐다**

앞으로 WordPress 플러그인과 테마는 **MCP 도구(Tools)**로 기능을 노출해야 할 것입니다. AI 에이전트가 플러그인 기능을 자연어로 호출할 수 있도록 하는 것이 UX의 핵심이 됩니다.

**새로운 에이전시 서비스**

"AI 에이전트 워크플로우 설계"가 새로운 서비스가 될 수 있습니다. 고객의 블로그를 분석하고, 적절한 MCP 도구와 자동화 시나리오를 구성하는 컨설팅이죠.

### 블로거 관점

**콘텐츠 제작의 병목 제거**

글 쓰는 것은 좋아하지만 SEO, 카테고리 정리, 이미지 최적화, 댓글 관리 같은 **잡무**가 부담이었다면, 이제 AI 에이전트에게 맡길 수 있습니다.

**다국어 블로그 운영의 현실화**

AI 에이전트가 한국어 포스트를 작성하면, 같은 에이전트에게 "이걸 영어로 번역해서 `/en/` 카테고리에 포스트 만들어줘"라고 지시할 수 있습니다.

**콘텐츠의 질과 양 동시 확보**

사람은 핵심 아이디어와 전략에 집중하고, AI 에이전트가 그것을 여러 형식(블로그 포스트, 랜딩 페이지, FAQ, 소셜 미디어 스니펫)으로 전개합니다.

## 시작하기

WordPress.com의 **모든 유료 플랜**에서 MCP 쓰기 기능을 사용할 수 있습니다:

1. `wordpress.com/me/mcp`에서 MCP 활성화
2. 사용할 쓰기 기능 토글 ON
3. AI 클라이언트 연결 (Claude Desktop, Cursor, ChatGPT, OpenClaw 등 MCP 지원 도구)
4. 자연어 명령으로 작업 시작

자세한 기능 목록은 [MCP Tools Reference](https://developer.wordpress.com/docs/mcp/tools-reference/)를 참고하세요.

## CMS의 미래 — AI-Native 퍼블리싱

WordPress.com의 MCP 통합은 단순한 기능 추가가 아닙니다. **CMS의 패러다임 전환**입니다.

과거 CMS는 "사람이 웹 페이지를 쉽게 만들도록 돕는 도구"였습니다.  
이제 CMS는 "AI 에이전트와 사람이 협업하는 플랫폼"으로 진화하고 있습니다.

**WordPress가 웹의 43%를 차지**한다는 것을 기억하세요. 이 변화는 인터넷 콘텐츠 생태계 전체를 바꿀 수 있습니다.

앞으로 우리는 질문을 던져야 합니다:  
- 웹 콘텐츠의 대부분이 AI가 작성한다면, 검색 엔진 최적화는 어떻게 바뀔까?
- 사람이 쓴 글과 AI가 쓴 글을 구분해야 할까? 구분할 수 있을까?
- 블로거의 역할은 "작가"에서 "편집장 겸 전략가"로 바뀌는 걸까?

**AI 에이전트가 블로그를 쓰는 시대**는 이미 시작되었습니다. 당신은 어떻게 대응하시겠습니까?
